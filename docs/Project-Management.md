# Project Management

## Labels

Recommended GitHub labels:

- `type:feature`
- `type:bug`
- `type:docs`
- `type:infra`
- `type:simulation`
- `priority:p0` / `priority:p1` / `priority:p2`
- `status:blocked`
- `status:ready`

## Issue types

- Feature: new capability or integration
- Bug: regression or incorrect behavior
- Docs: documentation improvements
- Simulation: SITL/mission demo regressions

## Definition of Done (DoD)

- Unit tests added/updated
- `make test` and `make lint` pass
- Docs updated as needed
- Safety gates reviewed if autonomy logic changes
